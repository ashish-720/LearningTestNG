TestNG Annotations

Configuration annotations that control the setup and teardown lifecycle of your test suite — from suite-wide initialization down to per-method cleanup.

Execution order — outermost to innermost
Suite scope
@BeforeSuite Runs once
Test (XML <test>) scope
@BeforeTest Per <test> tag
Class scope
@BeforeClass Per class
Method scope
@BeforeMethod Before each @Test
@Test method runs Your test logic
@AfterMethod After each @Test
@AfterClass Per class
@AfterTest Per <test> tag
@AfterSuite Runs once

Click any annotation above to jump to its reference card.

Suite

@BeforeSuite

Runs once before the entire suite. Use for global setup like DB connections, config loading, report initialization.

Suite

@AfterSuite

Runs once after all tests in the suite complete. Use for global teardown — closing DB pools, generating reports.

Test tag

@BeforeTest

Runs before any test method in the XML <test> tag. Useful for browser launch, environment setup per test group.

Test tag

@AfterTest

Runs after all test methods in the XML <test> tag complete. Use for browser quit, cleanup per test group.

Class

@BeforeClass

Runs once before the first method in the current class. Page Object init, shared state setup per class.

Class

@AfterClass

Runs once after all test methods in the current class. Release class-level resources, log class results.

Method

@BeforeMethod

Runs before each individual @Test method. Open browser, navigate to base URL, log in — fresh state per test.

Method

@AfterMethod

Runs after each individual @Test method. Take screenshots on failure, close browser, clear cookies.

BaseTest.java — all 8 annotations
package tests;

import org.testng.annotations.*;
import org.openqa.selenium.WebDriver;
import org.openqa.selenium.chrome.ChromeDriver;

public class BaseTest {

    protected WebDriver driver;

    // ─── Suite scope (runs ONCE for the whole XML suite) ─────────────────

    @BeforeSuite
    public void globalSetup() {
        System.out.println("[BeforeSuite] Loading config, setting up test DB...");
        ConfigLoader.load("qa.properties");
        ReportManager.init();
    }

    @AfterSuite
    public void globalTeardown() {
        System.out.println("[AfterSuite] Generating Extent Report, closing DB pool...");
        ReportManager.flush();
        Database.closePool();
    }

    // ─── Test-tag scope (runs once per <test> in testng.xml) ─────────────

    @BeforeTest
    public void browserSetup() {
        System.out.println("[BeforeTest] Launching Chrome...");
        driver = new ChromeDriver();
        driver.manage().window().maximize();
        driver.get(ConfigLoader.get("base.url"));
    }

    @AfterTest
    public void browserTeardown() {
        System.out.println("[AfterTest] Closing browser...");
        if (driver != null) driver.quit();
    }

    // ─── Class scope (runs once per test class) ───────────────────────────

    @BeforeClass
    public void classSetup() {
        System.out.println("[BeforeClass] Initialising Page Objects for "
            + getClass().getSimpleName());
    }

    @AfterClass
    public void classTeardown() {
        System.out.println("[AfterClass] Releasing class-level resources...");
    }

    // ─── Method scope (runs before/after EVERY @Test) ─────────────────────

    @BeforeMethod
    public void methodSetup() {
        System.out.println("[BeforeMethod] Navigating to login page, clearing cookies...");
        driver.manage().deleteAllCookies();
        driver.navigate().to(ConfigLoader.get("base.url"));
    }

    @AfterMethod
    public void methodTeardown(ITestResult result) {
        if (result.getStatus() == ITestResult.FAILURE) {
            System.out.println("[AfterMethod] Test FAILED → taking screenshot...");
            ScreenshotUtil.capture(driver, result.getName());
        }
        System.out.println("[AfterMethod] Test '" + result.getName() + "' done.");
    }
}
LoginTest.java — inherits BaseTest
public class LoginTest extends BaseTest {

    @Test
    public void validLogin() {
        driver.findElement(By.id("txtUsername")).sendKeys("Admin");
        driver.findElement(By.id("txtPassword")).sendKeys("admin123");
        driver.findElement(By.id("btnLogin")).click();
        Assert.assertTrue(driver.getTitle().contains("Dashboard"));
    }

    @Test
    public void invalidLogin() {
        driver.findElement(By.id("txtUsername")).sendKeys("wrong");
        driver.findElement(By.id("txtPassword")).sendKeys("wrong");
        driver.findElement(By.id("btnLogin")).click();
        Assert.assertTrue(driver.getPageSource().contains("Invalid credentials"));
    }
}

Key insight: @BeforeTest / @AfterTest correspond to the <test> XML tag — not to individual @Test methods. This is the most common source of confusion.
Annotation Scope How many times? Typical use
@BeforeSuite Suite Once — very first Config load, DB pool, report init
@BeforeTest Test tag Once per <test> in XML Launch browser, set environment
@BeforeClass Class Once per test class Page Object init, shared data
@BeforeMethod Method Before EVERY @Test method Navigate to URL, delete cookies, login
@Test Test run Each test method Your actual test assertions
@AfterMethod Method After EVERY @Test method Screenshot on fail, logout, clear state
@AfterClass Class Once per test class Release class-level resources
@AfterTest Test tag Once per <test> in XML Quit browser, cleanup per group
@AfterSuite Suite Once — very last Flush reports, close DB, send email
Quick-reference: alwaysRun attribute
// Runs even if a dependency method failed
@AfterMethod(alwaysRun = true)
public void safeTeardown() {
    // This will run even if @BeforeMethod threw an exception
    if (driver != null) driver.quit();
}
Quick-reference: inheritance
// Parent class
public class BaseTest {
    @BeforeMethod
    public void setup() { /* runs for ALL child classes */ }
}

// Child class — parent's @BeforeMethod runs FIRST
public class LoginTest extends BaseTest {
    @BeforeMethod
    public void extraSetup() { /* runs AFTER parent's setup() */ }
}
