Maven + TestNG + Surefire Quick Reference

1. pom.xml Basics
- groupId, artifactId, version identify the project.
- <properties> can define versions: testng.version, selenium.version, etc.
- <dependencies> add libraries: TestNG, Selenium, WebDriverManager, Apache POI.

2. Maven Surefire Plugin + TestNG
- Surefire runs tests during `mvn test`.
- Configure TestNG suites via:
  <plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-surefire-plugin</artifactId>
    <version>3.2.5</version>
    <configuration>
      <suiteXmlFiles>
        <suiteXmlFile>pack1_CreateFirstTestNGTest.xml</suiteXmlFile>
      </suiteXmlFiles>
    </configuration>
  </plugin>

3. testng.xml Purpose
- Controls which tests run, in what order, and how:
  * Select specific classes/methods.
  * Use groups (smoke, regression).
  * Run in parallel.
  * Pass parameters.
- Basic structure:
  <!DOCTYPE suite SYSTEM "http://testng.org/testng-1.0.dtd">
  <suite name="MySuite">
    <test name="Smoke">
      <classes>
        <class name="com.example.LoginTest"/>
      </classes>
    </test>
  </suite>

4. Multiple testng.xml Files
- Example files:
  smoke.xml
  regression.xml
- Master suite that includes both:
  <suite name="Master">
    <suite-files>
      <suite-file path="smoke.xml"/>
      <suite-file path="regression.xml"/>
    </suite-files>
  </suite>

5. Running Suites
- Default (from pom.xml):
  mvn test
- Override from command line (Surefire property):
  mvn test -Dsurefire.suiteXmlFiles=pack2_GenerateTestNGXmlFile.xml
- Multiple suites:
  mvn test -Dsurefire.suiteXmlFiles=pack1_CreateFirstTestNGTest.xml,pack2_GenerateTestNGXmlFile.xml

6. Making pom.xml Flexible
- Use a property with default:
  <suiteXmlFiles>
    <suiteXmlFile>${suiteXmlFile:pack1_CreateFirstTestNGTest.xml}</suiteXmlFile>
  </suiteXmlFiles>
- Then run:
  mvn test -DsuiteXmlFile=pack2_GenerateTestNGXmlFile.xml

7. JaCoCo (Optional)
- Measures code coverage of your tests.
- Typical config:
  <plugin>
    <groupId>org.jacoco</groupId>
    <artifactId>jacoco-maven-plugin</artifactId>
    <version>0.8.12</version>
    <executions>
      <execution>
        <goals>
          <goal>prepare-agent</goal>
        </goals>
      </execution>
      <execution>
        <id>report</id>
        <phase>test</phase>
        <goals>
          <goal>report</goal>
        </goals>
      </execution>
    </executions>
  </plugin>

8. Maven Compiler Plugin (Optional but Recommended)
- Ensures correct Java version (e.g., 8) and compiler options:
  <plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-compiler-plugin</artifactId>
    <version>3.13.0</version>
    <configuration>
      <source>8</source>
      <target>8</target>
    </configuration>
  </plugin>

9. mvn test — What Actually Happens
   When you run mvn test in your project specifically:
   mvn test
       │
       ├─ 1. validate     → pom.xml looks fine ✓
       │
       ├─ 2. compile      → compiles src/main/java (your Page Objects,
       │                    utilities, base classes etc.)
       │
       ├─ 3. test-compile → compiles src/test/java (your @Test classes)
       │
       └─ 4. test         → Surefire plugin kicks in
                                │
                                └─ reads your testng.xml
                                        │
                                        └─ runs all @Test methods defined there
                                                │
                                                ├─ Passes ✓ → BUILD SUCCESS
                                                └─ Fails  ✗ → BUILD FAILURE

10. What is the /target folder?
    Every time Maven compiles or packages, output goes into /target:
    target/
      ├── classes/          ← compiled main code (.class files)
      ├── test-classes/     ← compiled test code (.class files)
      ├── surefire-reports/ ← test results (XML + HTML reports)
      └── LearningTestNG-1.0-SNAPSHOT.jar  ← produced by mvn package

11. Quick Reference Card
mvn validate          → just check pom.xml
mvn compile           → compile main code only
mvn test-compile      → compile main + test code
mvn test              → compile everything + run tests  ← you'll use this most
mvn clean test        → delete target + run tests fresh ← safest option
mvn package           → compile + test + create JAR
mvn install           → package + save to local .m2 cache
mvn clean install     → most common full build command

12. In Case you want to run the test from terminal. You need maven downloaded and java installed and corresponding HOME path set
Open Command Prompt and run:
where mvn
If it returns nothing, Maven is not on PATH. Also check:
java -version
Java must be installed first — Maven needs it.
Step 2 — Download Maven
Go to: https://maven.apache.org/download.cgi
Download the Binary zip archive — something like:
apache-maven-3.9.x-bin.zip
Step 3 — Extract It
Extract to a simple path like:
C:\Program Files\Apache\maven
Avoid spaces and deep nesting. After extraction you should see:
C:\Program Files\Apache\maven\
    ├── bin\
    │    └── mvn.cmd   ← this is what CMD calls
    ├── lib\
    └── conf\
Step 4 — Set Environment Variables
This is the most important step. You need to set two things:
4a. Create MAVEN_HOME

Search "Environment Variables" in Windows Start
Click "Edit the system environment variables"
Click "Environment Variables" button
Under System Variables → click New
Variable name:  MAVEN_HOME
Variable value: C:\Program Files\Apache\maven

4b. Add to PATH

In the same System Variables section
Find and select Path → click Edit
Click New and add:
%MAVEN_HOME%\bin

Step 5 — Verify
Close your current CMD window completely (old window won't pick up new PATH), open a fresh one and run:
mvn -version
You should see something like:
Apache Maven 3.9.x
Maven home: C:\Program Files\Apache\maven
Java version: 1.8.x, vendor: ...
